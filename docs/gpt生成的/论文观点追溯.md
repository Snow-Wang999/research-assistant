# 一、这个需求的本质是什么？

你问的不是“能不能”，而是：

> 当系统给出一个结论时，用户能否看到：
>
> * 这句话来自哪篇论文
> * 来自论文中的哪一段
> * 哪一句原文支持了这个结论

本质上，这是：

> **结论 → 证据 → 原文句级映射**

这是学术研究中最核心的可信链条。

---

# 二、从硕博用户视角：这个需求有多重要？

对硕博用户来说：

| 没有句级追溯 | 有句级追溯 |
| ------ | ----- |
| 不敢直接用  | 可以验证  |
| 只能参考   | 可以引用  |
| 需要重新找  | 可直接定位 |
| 心理不信任  | 学术安全感 |

**硕博用户真实心理是：**

> AI 给的结论我不敢信，除非我能看到它来自哪一句原文。

所以这个能力不是“锦上添花”，而是：

> 是否敢在论文中使用你产品输出的分水岭。

---

# 三、从产品定位上看：这是护城河能力

目前大多数产品：

* 给结论
* 给论文链接
* 但不告诉你“是哪句话”

这导致：

> 引用可信度仍然断裂。

如果你能做到：

```
结论A
 → 支持句1（Paper X, Section 3, Sentence 2）
 → 支持句2（Paper Y, Section 4, Sentence 1）
```

你会直接进入：

> 学术研究工具第一梯队能力范畴

---

# 四、这个能力对 Deep Research 产品的意义

你现在产品的核心是：

> Deep Research = 结构化认知 + 可验证证据

那么：

* 没有句级追溯 → 只是“结构化总结”
* 有句级追溯 → 才是“研究级证据系统”

所以：

> 这个能力是 Deep Research 产品闭环成立的关键一环。

---

# 五、从技术实现上，它是否可行？

### 是可行的，而且并非幻想级难度。

基本技术路径：

```
原文切分 → 段落向量化 → 句子向量化
        ↓
结论 embedding
        ↓
相似度匹配
        ↓
Top-K 支持句
```

再通过 LLM 验证：

> 该句是否真实支持该结论（Entailment 检测）

你可以实现为：

* 支持句检索
* 支持句验证
* 支持句排序
