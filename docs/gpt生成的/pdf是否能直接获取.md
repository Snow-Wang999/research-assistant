# 我的问题：
我问一个问题，你认为通过网络可以直接获取pdf的每一个段落吗
这是一个很关键的技术问题。直接回答：大多数情况下不能。

#claude code：
各数据源情况
来源	能否获取全文段落	说明
arXiv	⚠️ 间接可以	可下载 PDF，需本地解析
OpenAlex	❌ 不能	只有 metadata + 摘要
Semantic Scholar	⚠️ 有限	S2ORC 数据集有全文，但需下载 TB 级数据
PubMed Central	✅ 可以	开放获取，有 XML 全文 API
付费期刊	❌ 不能	需要机构订阅
技术路径

方案 A: 下载 PDF → 本地解析（可行）
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ arXiv PDF   │ → │ PyMuPDF     │ → │ 段落/句子    │
│ 下载        │    │ 解析        │    │ 切分        │
└─────────────┘    └─────────────┘    └─────────────┘

方案 B: API 直接获取（受限）
- PubMed Central：生物医学领域可行
- Semantic Scholar Full Text API：有配额限制
结论
"直接获取段落"基本不可行 - 没有通用 API 支持
arXiv 可以间接实现 - 下载 PDF + 本地解析
这正是为什么 v0.4.0 本地 PDF 库是前置条件
如果想做句级追溯，最现实的路径是：

用户上传 PDF（v0.4.0）
或自动下载 arXiv PDF 并解析
